PYTHON SETUP / VENV

まず、このPCでPythonを一度動かす

スクリプトや仮想環境の話に入る前に、入口がひとつ抜けがちです。「自分のPCでPythonが動くか」「短い .py を一度実行できるか」。ここを越えておくと、この先の自動化がぐっと楽になります。「環境構築」と身構えず、まず一度動かすことが目標です。

今回の着地点

新しいフォルダで仮想環境を作り、短いプログラムを実行し、終了後にもう一度同じ手順を再現できる。

入っているか確認する（python か py か）

まずPythonがこのPCにあるかを確かめます。Windowsでは python と py のどちらで動くかが環境によって違うので、両方試します。バージョンが表示された方が、あなたのPCで使えるコマンドです。

python --version
py --version

どちらも表示されなければ、AIに「Windowsです。Pythonが入っていないようなので入れ方を教えて」と頼めば案内してくれます。

.py ファイルを作って、ターミナルから実行する

テキストエディタで hello.py を作り、一行だけ書きます。

print("動いた")

そのファイルがあるフォルダで、確認できた方のコマンドで実行します。動いた と表示されれば、最初の成功体験です。

python hello.py

仮想環境を作って有効化する

仮想環境は、そのフォルダ専用の「Pythonの作業部屋」です。部品（ライブラリ）を入れても、PC全体や他のプロジェクトと混ざりません。新しいフォルダで作り、有効化します。

  1. 仮想環境を作る：python -m venv .venv
  2. 有効化する（Windows）：.venv\Scripts\activate
  3. 使い終わったら無効化：deactivate

有効化されると、行の先頭に (.venv) のような表示が出ます。これが「いま作業部屋の中にいる」目印です。

pip は「今いる環境に部品を足す」道具

pip は、いま有効化しているPython環境へ外部の部品を追加するコマンドです。仮想環境を有効化してから pip install すれば、その部品はこのフォルダの作業部屋にだけ入ります。だから他に影響しません。

エラーが出たら、何をAIへ渡すか

つまずいたら、推測で直す前に次をそのまま渡すと、原因を絞りやすくなります。

  • 使ったコマンド（python か py か）
  • 表示されたエラーメッセージ全文
  • OS（Windows）と、いま仮想環境を有効化しているか

そのまま使える依頼文

Pythonを自分のPCで初めて動かします。Windowsを想定して、python と py のどちらで動くかの確認、新しいフォルダの作成、仮想環境の作成と有効化、短い hello.py の作成と実行、終了（無効化）までを順番に案内してください。pip が「今使っている環境に部品を足すもの」である点も一言で説明してください。

押さえるべきこと

  • まず python か py か、自分のPCで動く方を確かめる。
  • 短い .py を一度実行できれば、最初の入口は越えている。
  • 仮想環境はフォルダ専用の作業部屋。部品が混ざらない。
  • pip は「今いる環境に部品を足す」道具。有効化してから使う。

やってみよう

新しいフォルダを作り、python -m venv .venv で仮想環境を作って有効化します。print("動いた") だけの hello.py を実行し、表示を確認したら deactivate で抜けます。最後に、もう一度同じ手順を最初から再現できるか試してみてください。
