Data processing

入力を読んで、変換して、安全に保存する

大量のデータをまとめて処理したいとき、画面は要りません。入力・変換・出力の3つに分けて考えると、最短で組めて、壊れにくくなります。

この単元でできるようになること
  • 処理を「入力・変換・出力」に分けて考えられる。
  • 途中経過と失敗した理由を記録して、後から原因を追える。

処理は、入力・変換・出力の3段で考える

「CSVを集計して」と一息に頼むと、どこで失敗したのか分からなくなります。最初に3つの段に分けておくと、問題が起きた場所がすぐ特定できます。

  1. 入力：元データを読む（ここでは絶対に書き換えない）
  2. 変換：集計や整形をする（途中の作業はコピーの上で）
  3. 出力：結果を保存する（最後にまとめて書き出す）

特に大切なのは、入力の元データを処理の途中で上書きしないことです。元が無事なら、失敗しても何度でもやり直せます。

元データを壊さない「置き換え方式」

1万行のような大きなデータでは、途中で止まる可能性を最初から織り込みます。直接本ファイルへ書き込むのではなく、一時ファイルに書いてから、全部うまくいったときだけ本ファイルへ差し替えます。

  1. 先に1〜2行だけ試して、変換のルールが正しいか検証する
  2. 結果は一時ファイルへ書き出す（本ファイルにはまだ触らない）
  3. 全行が成功したら、一時ファイルを本ファイルへ置き換える

途中で止まっても、本ファイルは手付かずのまま。「開始前の状態へいつでも戻れる」状態を、設計で先に確保しておきます。

直書きで止まる 元のCSVに直接上書き → 5000行目でエラー → 半分書き換わったファイルだけが残る。
置き換え方式 一時ファイルへ書き出し → 全成功なら差し替え。失敗しても元データは無傷。

失敗は止めずに、行ごとに記録する

1行おかしいだけで全部を止めると、最後の1行まで進めません。エラーが出た行はスキップして記録に残し、処理は続ける。終わった後で「どの行が、なぜダメだったか」を見返せるようにします。

そのまま使える依頼文

入力を読んで変換して保存する流れにしてください。途中で失敗しても元データは壊さず、一時ファイルに書いてから全成功時だけ本ファイルへ置き換える形でお願いします。行ごとのエラーは記録に残して、処理は止めずに続けてください。

押さえるべきこと

  • 処理は入力・変換・出力に分け、入力の元データは絶対に上書きしない。
  • 先に検証 → 一時ファイル → 全成功時のみ本ファイルへ置き換える。
  • 失敗した行は記録して飛ばし、途中で全体を止めない。

やってみよう

1万行のCSVを安全に変換する流れを、AIと一緒に設計してみましょう。次の文をそのまま渡してみてください。

1万行のCSVを別の形式に変換するスクリプトを設計してください。元データは壊さず、一時ファイルに書いてから全成功時だけ本ファイルへ置き換える方式にし、行ごとのエラーをログに残してください。初めての人向けに、確認手順も説明してください。
