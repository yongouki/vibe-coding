Static Hosting

作ったものを、URLで見せたい

ブラウザだけで動くものは、無料で誰でも開ける場所に置けます。サーバーの契約は要りません。手元のファイルを、URLを共有するだけで見せられる状態にします。

今回の着地点

HTML/CSS/JSをそのまま公開でき、URLを共有すれば誰でも開ける状態を作れるようになる。

バックエンドが無いなら、サーバー契約は要らない

ここまでに作ってきた画面は、ブラウザの中だけで動いています。サーバー側で計算したりデータベースに書き込んだりする処理（バックエンド）がありません。こういうものは 静的ホスティング という仕組みに置くだけで公開できます。

「静的」とは、置いたファイルをそのまま配るだけ、という意味です。常時動くサーバーを借りる必要も、月額を払う必要もありません。多くは無料で使えます。

過剰になりがち ブラウザだけで動くページなのに、公開のためにサーバーを契約して常時動かそうとする。
これで足りる ファイルを静的ホスティングへ置き、発行されたURLを共有して見てもらう。

GitHub Pagesは「一例」であって唯一解ではない

静的ホスティングの代表として、この単元では GitHub Pages を実習に使います。ただし、これだけが正解ではありません。ほかにも Netlify、Cloudflare Pages、Vercel など、同じように静的なファイルを無料で公開できるサービスがあります。

違いは、操作画面や連携方法の好み程度で、最初のうちは大きくありません。だから「みんなが使っているから」で1つ選び、まず一度URLで開けるところまで通すのが近道です。乗り換えは後からでもできます。

AIへ頼むときは「手順を、初めての人向けに」

これはブラウザだけで動くので、GitHub Pagesなどの静的ホスティングで公開する手順を、初めての人向けに教えてください。GitHub Pagesだけが唯一の選択肢ではないことを明記し、他の候補との違いを短く説明してから、リポジトリ作成からURLで開けるまでを案内してください。

「初めての人向けに」「リポジトリ作成からURLで開けるまで」と範囲を区切ると、AIが手順を飛ばさず順を追って説明してくれます。「唯一の選択肢ではない」と添えるのは、過剰に1つへ縛られないためです。

ここだけ持ち帰る

  • バックエンドが無いアプリは、静的ホスティングに置くだけで公開できる。
  • 多くは無料。常時動くサーバーの契約は要らない。
  • GitHub Pagesは便利な一例。唯一解ではなく、後から乗り換えられる。
