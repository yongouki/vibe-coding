WEB BASICS / HTML・CSS・JS

画面を作る前に、Webページの正体を知る

コードを自分で書けるようになる必要はありません。でも、AIが書いたものが「何でできているか」だけは先に知っておくと、この先ずっと迷いません。Webページは、たった3つの役割でできています。

今回の着地点

AIが出したコードを見て、HTML・CSS・JavaScriptのどれが何をしているかを役割で読み分けられる。

Webページは3つの役割でできている

どんなに複雑なサイトも、つきつめると次の3つの組み合わせです。家にたとえると分かりやすいです。

  • HTML：中身・構造。「ここに見出し」「ここにボタン」「ここに一覧」と、何があるかを決める。家でいう骨組みと間取り。
  • CSS：見た目。色・余白・大きさ・並び方など、どう見えるかを決める。家でいう内装や塗装。
  • JavaScript：動き。ボタンを押したら表示が変わる、入力に反応する、など押したらどうなるかを決める。家でいうスイッチや仕掛け。

この3つの役割が分かっていれば、AIが書いた長いコードも「これは中身、これは見た目、これは動き」と仕分けながら読めます。

ブラウザが、その3つを「画面」に変えている

HTML・CSS・JavaScriptは、それ自体はただの文字（テキスト）です。その文字を読み取って、実際の画面として表示してくれているのがブラウザ（Chrome、Edge、Safariなど）です。

つまり、あなたが見ている「ボタンのあるきれいな画面」は、ブラウザが文字を解釈した結果です。コードを書く＝魔法の呪文ではなく、ブラウザへの指示書を文字で書いている、と捉えると気が楽になります。

身構えてしまう 「プログラミング＝才能が要る難しいもの」と思って、最初の一歩が出ない。
正体が分かる 「コードは文字。AIが書き、自分は役割で読んで指示する」と分かり、安心して触れる。

「ブラウザの中」と「ブラウザの外」── フロントエンドとバックエンド

ここまでの HTML・CSS・JavaScript は、すべてブラウザの中で動く部分です。この、目に見える画面側をまとめてフロントエンドと呼びます。

一方で、「みんなで同じデータを共有したい」「秘密の鍵（APIキーなど）を人に見られないよう隠したい」となると、ブラウザの中だけでは足りず、ブラウザの外にもう一つ動く場所が要ります。この目に見えない裏側をバックエンドと呼びます。

フロントエンド ブラウザの中で動く、目に見える画面側。HTML・CSS・JavaScript はここ。
バックエンド ブラウザの外で動く裏側。共有データの保存や、秘密の鍵を扱うのはここ。

今の段階はフロントエンドだけで十分です。バックエンドは「自分の端末だけ」を超えて共有したくなった時に出てきます。今は名前と場所——中がフロント、外がバック——だけ覚えておけば大丈夫です。

あなたの役割は「書く」より「読んで指示する」

バイブコーディングでは、コードの多くをAIが書きます。だから、あなたに必要なのは全部を覚えることではありません。

  • AIが出したものを、3つの役割で読めること
  • 「ここの動きを変えて」「見た目だけ直して」と、役割を指して頼めること

用語を暗記するより、「これはHTML・CSS・JavaScriptのどれ？」とAIに聞き返せる方が、ずっと役に立ちます。

そのまま使える依頼文

プログラミングは初めてです。1ファイルだけの、ボタンを押すと文字が変わるごく小さなWebページを作ってください。そのうえで、どこがHTML（中身・構造）、どこがCSS（見た目）、どこがJavaScript（動き）かを、コードにコメントを付けて初心者向けに説明してください。作ったファイルをブラウザで開く方法も教えてください。

ここだけ持ち帰る

  • Webページは HTML（中身）・CSS（見た目）・JavaScript（動き）の3つでできている。
  • コードはただの文字で、ブラウザがそれを画面に変えている。
  • ブラウザの中＝フロントエンド、ブラウザの外＝バックエンド。今はフロントだけで十分。
  • 全部覚える必要はない。役割で読み分け、役割を指して頼めれば十分。

5分だけやってみる

上の依頼文をAIにそのまま渡して、小さなページを1つ作らせてください。返ってきたコードを上から眺め、「ここは中身（HTML）」「ここは見た目（CSS）」「ここは動き（JavaScript）」と、3色に塗り分けるつもりで指でなぞってみましょう。全部分からなくて大丈夫。3つの固まりが見分けられれば成功です。
