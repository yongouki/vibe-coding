Python

小さなスクリプトを書いて、同じ手順を任せる

手で繰り返している作業は、短いプログラムにすると一発で終わります。大事なのは、書いたものが別のPCでも同じように動くことです。

この単元でできるようになること
  • 変数・条件分岐・関数を使って、簡単な処理を組み立てられる。
  • 使ったライブラリと手順を、別のPCでも再現できる形で残せる。

「今回だけ動く」と「いつでも動く」は別物

自分のPCで動いたから完成、ではありません。スクリプトは、外から持ってきた部品（ライブラリ）に頼って動いていることが多く、その部品が無いPCでは止まります。

だから、コードと一緒に「どの部品が必要か」「どう実行するか」を残しておきます。Pythonでは、必要な部品の一覧を requirements.txt というファイルに書き出しておくのが定番です。これがあると、別のPCでも同じ状態を一行で作り直せます。

  1. スクリプト本体（やりたい処理）
  2. 必要なライブラリの一覧（requirements.txt）
  3. 実行手順（どう動かすかのメモ）

AIには「再現できる形で」と一言足す

ただ「Pythonで書いて」だと、自分のPCでしか動かないコードになりがちです。再現性まで頼むと、必要なファイルと手順がセットで返ってきます。

そのまま使える依頼文

これはPythonの短いスクリプトでお願いします。必要なライブラリと実行手順も、別のPCで再現できる形（requirements.txt と起動コマンド）で書いてください。Pythonは初めて触ります。

手元でしか動かない コードだけ受け取る → 別のPCで実行したら「ライブラリが無い」で止まる。
どこでも再現できる コード＋requirements.txt＋手順を受け取る → 別のPCでも一行で同じ状態を作れる。

長い1本より、小さな関数に分ける

処理が長くなってきたら、意味のまとまりごとに関数へ切り分けます。名前を付けて分けておくと、後で読み返したときに「どこが何をしているか」がすぐ分かり、直したい場所だけ直せます。

  • 変数：途中の値に名前を付けて置いておく場所
  • 条件分岐：「もし◯◯なら」で処理を変える
  • 関数：一つの役割をまとめて、名前で呼び出せるようにする

用語を覚えるより、「この処理に名前を付けて分けて」とAIに頼める方が役に立ちます。

押さえるべきこと

  • 動いた＝完成ではない。別のPCでも同じ状態を作れるかまで確認する。
  • コードと一緒に、必要なライブラリ（requirements.txt）と手順を残す。
  • 長い処理は、役割ごとに名前を付けた関数へ分ける。
