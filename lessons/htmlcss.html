HTML / CSS

まず、触れる画面を一枚出す

HTMLは画面に何があるか、CSSはそれをどう見せるか。最初からきれいなサイトを目指さず、内容が読めて、押す場所が分かる一枚を作ります。

今回の着地点

見出し、説明、入力欄、ボタンを置き、ブラウザで開いて構造と見た目を別々に直せるようになる。

HTMLは「何があるか」を決める

支出記録の最初の画面なら、必要なのは多くありません。

  • ページの見出し
  • 金額を入れる入力欄
  • メモを入れる入力欄
  • 追加ボタン
  • 追加した内容を出す一覧

この役割をHTMLで表します。見出しは見出し、入力は入力、操作はボタンとして書く。最初からタグを全部覚える必要はありません。AIが出したHTMLを見て、「これは何の役割か」が読めれば先へ進めます。

<main>
  <h1>支出メモ</h1>
  <label>
    金額
    <input type="number">
  </label>
  <button type="button">追加する</button>
  <ul></ul>
</main>

まだボタンは動きません。それで構いません。まず「どこに入力し、どこを押し、結果がどこへ出るか」が画面から分かるかを見ます。

CSSは「どう見せるか」を決める

HTMLを変えずに、余白、幅、色、並び方を調整します。最初に効くのは派手な装飾より、次の3つです。

  1. 画面の端にくっつかない余白
  2. 入力欄とボタンの幅をそろえる
  3. 見出し、説明、操作の間を空ける
先にやりすぎる 動きが決まる前に、アニメーションや細かな配色を何度も調整する。
最初はここまで スマホ幅で横にはみ出さず、入力欄と追加ボタンを迷わず見つけられる。

意味のある要素を使うと、後で楽になる

クリックできる見た目をすべて div で作ることもできます。ただ、操作なら button、移動なら a を使った方が、キーボード操作や読み上げなど、ブラウザが持つ仕組みをそのまま使えます。

見た目だけボタン <div onclick="add()">追加</div>
役割もボタン <button type="button">追加</button>

入力欄には、見た目の近くに文字を置くだけでなく、label で名前を結びつけます。マウスを使わずTabキーで入力欄とボタンへ移動でき、今いる場所が見えることも、最初の画面で確かめたい基本です。

AIへ頼むときは、画面の役割を言う

1ファイルのHTMLで、支出メモの画面を作ってください。見出し、金額入力、メモ入力、追加ボタン、空の一覧を置きます。今回はJavaScriptを書かず、画面構造だけ作ります。スマホ幅で横スクロールしないようCSSを同じファイルへ入れ、各要素の役割を説明してください。

「今回はJavaScriptを書かない」と区切ることで、画面構造だけを確認できます。次の段階で、ボタンを押したときの動きを足します。

ここだけ持ち帰る

  • HTMLは画面の役割、CSSは見せ方を担当する。
  • 最初は、余白・幅・順番が分かれば十分。
  • ボタンやリンクは、見た目だけでなく役割に合った要素を使う。

15分だけ作る

好きな題材で、見出し、短い説明、ラベル付き入力欄、ボタンを一枚に置いてください。ブラウザで開き、スマホくらいの幅にして横へはみ出さないことを確認します。最後にTabキーだけで入力欄とボタンへ移動し、フォーカス位置が見えるかも確かめてください。
