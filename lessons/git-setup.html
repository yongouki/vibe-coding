GIT SETUP / FIRST COMMIT

Gitを始めて、最初の保存点をひとつ作る

一つ動くものができたら、まずやることは公開ではありません。いまの状態を「戻れる保存点」として手元に残すことです。ここがGitの出発点で、これさえできれば、この先どれだけ壊しても怖くなくなります。

今回の着地点

いま動いているフォルダで最初のコミット（保存点）を1つ作る。GitHubへの公開はまだしない。

GitとGitHubは別物

最初につまずきやすいのがこの2つの違いです。ざっくり言うと、

  • Git：自分のPCの中で、変更を保存点として積んでいく仕組み。ネットは要りません。
  • GitHub：そのGitの履歴を、ネット上に置いて公開・共有する場所。後から使えばよく、最初は要りません。

今回はGitだけを使います。アカウント登録もURL公開もまだ不要です。手元に保存点を作ることに集中します。

入っているか確認する

まずGitがこのPCに入っているかを確かめます。ターミナル（Windowsなら PowerShell）で次を実行し、バージョンが表示されれば入っています。

git --version

表示されなければ、AIに「Gitが入っていないようです。このOSへの入れ方を教えて」と頼めば案内してくれます。

フォルダでGitを始めて、最初の保存点を作る

作業フォルダの中で、次の流れを一度たどります。順番に意味があります。

  1. git init … このフォルダでGitを始める（最初の1回だけ）
  2. git status … いま何が保存対象になっているか、現在地を見る
  3. git add . … 保存したいものを保存対象に選ぶ
  4. git commit -m "最初の保存点" … 選んだ内容を保存点として残す

git status は、迷ったらいつでも実行してよい「現在地を見る」コマンドです。何かする前後に挟むと、状態がつかめます。

手作業でコピー 「念のため」とフォルダを日付付きで何個もコピーする → どれが最新か分からなくなる。
保存点で残す コミットとして残す → 一覧で履歴を追え、いつでもその時点に戻れる。

AIに最初の保存点づくりを手伝ってもらう

コマンドを覚えていなくても大丈夫です。「初めてで、公開はまだしない」と添えると、手元の作業だけに絞って案内してくれます。

そのまま使える依頼文

Gitを初めて使います。今この作業フォルダにあるものを、最初の保存点（コミット）として残したいです。GitとGitHubの違いを一言で説明し、Gitがこの環境に入っているかの確認、git init、git status での状態確認、git add と git commit までを、初めての人向けに順番に案内してください。GitHubへの公開は今回はまだしません。

ここだけ持ち帰る

  • GitはPC内の保存点づくり、GitHubはネットでの公開。今回はGitだけ。
  • 最初の一歩は公開ではなく、手元で最初のコミットを作ること。
  • git status はいつでも「現在地」を見るコマンド。迷ったら実行する。
  • add（選ぶ）→ commit（残す）で保存点ができる。

5分だけやってみる

いま動いているフォルダで git init を実行し、git status で状態を見てから、git add . と git commit -m "最初の保存点" で最初のコミットを作ってみましょう。最後に git log で、その保存点が残っていることを確認します。

チェックポイント

大事なのは、いきなり公開することではなく、手元に「戻れる保存点」を1つ作れたかどうかです。git log に最初のコミットが出ていればクリアです。
