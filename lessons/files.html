Files / CLI

ターミナルで、ファイルの場所を確かめてから動かす

ファイルがたくさんあると、どこに何があるか分からなくなります。まず「今どこにいるか」を確認する。それだけで、移動も実行も迷わなくなります。

この単元でできるようになること
  • 今いる場所と、対象ファイルがそこに実在するかを確認できる。
  • コマンドの結果を読んで、次に何をすればいいか自分で決められる。

「動かない」の半分は、場所のズレ

ターミナルでファイルを実行しようとして「そんなファイルは無い」と言われる。多くの場合、ファイルが壊れているのではなく、自分が思っている場所と、ターミナルが今いる場所がズレているだけです。

だから最初にやることは、いつも同じです。今いる場所を確認し、そこに対象ファイルが見えるかを確かめる。この2つだけで、つまずきの多くは消えます。

  1. 今いる場所を表示する（Windowsなら cd、Mac/Linuxなら pwd）
  2. その場所のファイル一覧を表示する（dir または ls）
  3. 対象が一覧に見えたら、初めて実行する

一覧に目的のファイルが無ければ、まだ実行する段階ではありません。先に正しいフォルダへ移動します。

AIには「今の場所の確認から」と頼む

コマンドを丸暗記する必要はありません。やりたいことと、初めてである点を伝えれば、順番に教えてくれます。

そのまま使える依頼文

ターミナルでこのフォルダのファイルを一覧・移動・実行したいです。まず今いる場所を確認するコマンドから、順番に教えてください。WindowsのPowerShellを使っていて、ターミナルは初めてです。

「初めてです」「PowerShellです」のように環境を一言添えると、自分の画面で動くコマンドが返ってきます。MacとWindowsで書き方が違うので、ここを省くと噛み合わないことがあります。

いきなり実行 コマンドをコピペして実行 → 「ファイルが無い」と言われ、原因が分からないまま手が止まる。
場所を確かめてから 今いる場所と一覧を表示 → 対象が見えるのを確認してから実行する。

エラーは、次の一手のヒント

赤い文字が出ても、慌てて閉じないでください。メッセージには、次に何をすればいいかが書かれています。そのまま読めなければ、AIに貼って聞けば十分です。

  • 「No such file（ファイルが無い）」 → 場所か名前のズレ。まず一覧で実在を確認する。
  • 「Permission denied（権限がない）」 → 場所は合っている。これは別の問題なので混ぜない。

大事なのは、この2つを切り分けることです。「無い」問題と「権限」問題を一緒に直そうとすると、原因が見えなくなります。

押さえるべきこと

  • 実行の前に、今いる場所と対象ファイルの実在を必ず確認する。
  • 「ファイルが無い」と「権限がない」は別の問題。混ぜて直さない。
  • AIには環境（OSやシェル）を一言添えて、今の場所の確認から頼む。

やってみよう

フォルダ内のファイル名を一覧で書き出す短いスクリプトを作りましょう。次の文をそのままAIに渡してみてください。

指定したフォルダの中のファイル名を一覧で表示する短いスクリプトを書いてください。実行方法と、対象フォルダの指定の仕方も、初めての人向けに説明してください。

チェックポイント
