ONE WORKING THING

まず、一つ動くものを完成させる

ここは、どの道を通ってきても合流する地点です。画面を出す、データを流す、認証を付ける、AIに評価させる──どれか一つでも実際に動いたなら、もう「作れる人」です。

今回の着地点

機能を増やす前に、一つ動くものを、自分にも他の人にも再現できる形で締めくくれるようになる。

どの枝から来ても、ゴールは同じ

このツリーには複数の道があります。ブラウザに画面を出す道、ファイルやデータを一括で処理する道、サーバーや認証を立てる道、AIの出力を確かめる道。どれも別々のことに見えますが、たどり着く先は一つです。

「決めた範囲のことが、実際に最後まで動く」。これが達成できていれば、通ってきた道に関係なく、あなたは一度ものを完成させたことになります。残りは、ここから少しずつ足していくだけです。

技術が多いほど偉い、ではない

作っていると、つい足したくなります。ログイン、データベース、外部API、きれいなデザイン。でも、いま欲しいものに対して技術が過剰だと、完成は遠のき、壊れたときに調べる範囲も広がります。

最初の軸は、「いま、他の人や他の端末が要るか」です。加えて、秘密情報を隠す必要があるか、大量データを扱うか、決まった時間に動かすか、止まると困るかも確認します。どれも無ければ、ブラウザの中だけで十分なことがほとんどです。

先に盛る 自分用のメモなのに、最初からログインとデータベースとデプロイをまとめて用意する。
いまに合わせる 自分・1端末・記録を残すだけなら、HTML / CSS / JS と localStorage で一度完成させる。

右側で一度動かしてから、「次に他の人にも使わせたい」と思った時に、はじめて公開や認証を足します。捨てているのではなく、順番を後ろに置いているだけです。

「動いた」を、再現できる状態にする

自分の画面で一度動いただけでは、まだ半分です。明日の自分や、見せたい相手が、同じ手順で同じ結果にたどり着けて、はじめて「完成」と言えます。最低限、次の3つがそろっていれば十分です。

  1. 動かし方：このファイルをブラウザで開く、など一行で書ける手順
  2. 成功の合図：何をすると、何が見えれば成功か
  3. 今回の範囲：何ができて、何はまだやらないか

そのまま使える依頼文

いま動いているこのページを、いったん完成として締めたいです。今の欲しいものに対して過剰にせず、最小構成のままにしてください。要らない技術は今は足さないでください。あわせて、開き方・成功の確認手順・今回やらないことを、短いメモにまとめてください。

よくある失敗

  • 完成間際に「ついでに」機能を足し始め、動いていたものまで壊す。
  • 自分の環境でしか動かず、手順を書かないので他の端末で再現できない。
  • まだ一人でしか使わないのに、公開や認証を先に組み込んで止まってしまう。

ここだけ持ち帰る

  • どの枝から来ても、ゴールは「決めた範囲が最後まで動く」ことで同じ。
  • 他の人・他の端末が要らないうちは、ブラウザの中だけで足りる。
  • 秘密・量・自動実行・止まったときの影響が増えたら、必要な技術を一つずつ足す。
  • 動かし方・成功の合図・今回の範囲を残し、再現できる形で締める。

5分だけやってみる

いま手元で動いているものを一つ選び、次の3行を書いてください。さらに「他人・他端末・秘密・量・自動実行・停止影響」の6項目を要／不要で判定し、不要な技術が紛れ込んでいないか差分を確認します。
