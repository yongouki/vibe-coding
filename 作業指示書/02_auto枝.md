## files — 大量のファイルを扱いたい
- ファイル: `lessons/files.html` ／ body: `lesson kind-auto`
- 前提: task（✅完成・lnav前に置く） ／ 次: python
- kicker: `Files / CLI`
- lead素材(desc): ファイルの場所を把握し、ターミナルから移動・実行・確認する。
- できるようになること(points): 現在地と対象ファイルを確認する／コマンドの結果を読んで次を決める
- AIへの言い方(SAY): 『ターミナルでこのフォルダのファイルを一覧／移動／実行したい。まず今いる場所を確認するコマンドから教えて』
- やってみよう(MISSION):
  - do: フォルダ内のファイル名を一覧で書き出すスクリプトを作ろう。
  - prompt: 「指定したフォルダの中のファイル名を一覧で表示する短いスクリプトを書いてください。実行方法と、対象フォルダの指定の仕方も初めての人向けに説明してください。」
- チェックポイント(quizヒント): 「ファイルが無い」問題と「権限」問題を混ぜない。まず現在地と対象の実在を確認する。

## python — 繰り返しを任せたい
- ファイル: `lessons/python.html` ／ body: `lesson kind-auto`
- 前提: files ／ 次: dataflow
- kicker: `Python`
- lead素材(desc): 小さなプログラムを書いて実行し、必要なライブラリを環境ごとに管理する。
- できるようになること(points): 変数・条件分岐・関数を使う／依存を再現できる形で残す
- AIへの言い方(SAY): 『これはPythonの短いスクリプトで。必要なライブラリと実行手順も、別のPCで再現できる形で書いて』
- やってみよう(MISSION):
  - do: 数字を入れると合計と平均を返す、小さなPythonスクリプトを作ろう。
  - prompt: 「Python で、数値のリストを受け取って合計と平均を表示する短いスクリプトを書いてください。実行方法と、自分の数字に差し替える箇所も説明してください。」
- チェックポイント(quizヒント): 「今回だけ動く」ではなく、別のPCでも同じ状態を作れるか（依存と手順の再現性）。

## dataflow — まとめて変換・集計したい
- ファイル: `lessons/dataflow.html` ／ body: `lesson kind-auto`
- 前提: python ／ 次: scheduler（または product へ分岐）
- kicker: `Data processing`
- lead素材(desc): 入力を読み、変換し、結果を保存する流れを組み立てる。画面が要らないなら最短で組める。
- できるようになること(points): 入力と出力を分けて考える／途中経過と失敗理由を記録する
- AIへの言い方(SAY): 『入力を読んで変換して保存する流れにして。途中で失敗しても元データは壊さず、行ごとのエラーを記録して』
- やってみよう(MISSION): （専用なし）SAYを土台に「1万行CSVを安全に変換する設計」を小課題化する。
- チェックポイント(quizヒント): 途中で止まっても開始前の状態へ戻れるか。先に検証→一時ファイル→全成功時のみ本ファイルへ置換、という堅牢設計。

## scheduler — 開かなくても勝手に動いてほしい
- ファイル: `lessons/scheduler.html` ／ body: `lesson kind-auto`
- 前提: dataflow ／ 次: 準備中
- kicker: `Scheduler / Cron / Webhook`
- lead素材(desc): 手で実行していた処理を、決まった時間や外部イベントをきっかけに自動で動かす。
- できるようになること(points): 同じ処理が重なっても壊れないようにする／実行結果と失敗を通知・記録する
- AIへの言い方(SAY): 『毎朝決まった時間に動かして。二度実行されても結果が壊れないように（冪等に）して、失敗時は通知して』
- やってみよう(MISSION):
  - do: 毎日1回の集計を、安全に再実行できるジョブとして設計しよう。
  - prompt: 「毎日決まった時間にCSVを集計するジョブを設計してください。CronまたはSchedulerを使い、重複実行への冪等性、実行履歴、失敗時の再試行と通知を含めてください。初心者向けの最小構成と確認方法も説明してください。」
- チェックポイント(quizヒント): 「一度しか呼ばれない」前提に依存しない。二度動いても同じ結果になるか（冪等性）。
