## 2. 【事前作業】着手前に1回だけ・担当1名が実施

データ上 `backend / database / auth` は `kind:"server"` だが、`lesson.css` に `kind-server` の色が無い。
**並行作業を始める前に**、誰か1名が以下を `lessons/lesson.css` に追記し、コミットしておく（ツリーの server 色 `#5b9dff` に合わせる）。

`:root{ … }` の色定義に1行追加：
```css
--server:#5b9dff;
```
`.kind-*` の定義行群に1つ追加：
```css
.kind-server{--accent:var(--server)}
```

これが終わるまで **AI・サーバー枝（03）は着手しない**。他の枝はこの事前作業と無関係に進めてよい。

## 3. ファイルの作り方（手順）

1. `lessons/_template.html` を複製し、`lessons/<ノードID>.html` にリネーム。
2. `<body class="lesson kind-XXX">` の `XXX` を、枝ファイル指定の kind に変える。
3. `<!-- 差し替え -->` を全部埋める。各枝ファイルの「ノード仕様」に素材がある。
4. 不要な任意ブロック（`compare` / `try` など）は、使わないなら削除してよい。
5. `LESSON_URLS` への登録は**自分ではやらず**、§4の手順で最後にまとめる。

### 各ノードに与えられる素材（枝ファイル内）
- **kicker（英語タグ）** … `<span class="kicker mono">` に入れる
- **h1 / lead** … 見出しと1〜2文要約。`name` と `desc` を土台に自然な日本語へ
- **できるようになること** … `goalbox` の箇条書き（pointsを噛み砕く）
- **AIへの言い方（SAY）** … そのまま「やってみよう」や本文の核に使える“使える文”
- **やってみよう（MISSION）** … 小さな実践課題＋AIへのコピペ依頼文（ある場合のみ）
- **チェックポイント** … ツリーのクイズへ繋ぐ一言。`hint` を参考に

## 4. LESSON_URLS 登録（統合担当が最後にまとめて）

各レッスンが書き上がったら、`vibe-skill-tree.html` の `LESSON_URLS`（321行目付近）に1行ずつ足す。
**衝突防止のため、個々人は追記せず、統合担当が完成分を一括登録する。**

```js
const LESSON_URLS={
  goal:"lessons/goal.html",
  collab:"lessons/collab.html",
  task:"lessons/task.html",
  htmlcss:"lessons/htmlcss.html",
  // ↓ 完成したノードIDをここに追記
  jsdom:"lessons/jsdom.html",
  // …
};
```
未登録のノードは、ツリー上で自動的にフォールバック表示になるだけで壊れない。だから**部分的に登録しても安全**。

## 5. 文章のトーン（重要）

- 専門用語は「使える文」の形で覚えさせる。用語の羅列・定義の暗記をさせない。
- 「こうなりがち（bad）／こう書く（good）」の対比を1組は入れると伝わりやすい（`compare` ブロック）。
- AIへの頼み方を必ず具体例で示す（SAY素材を活用）。
- 1ページ ≒ `goal.html` 程度（本文 h2 が 2〜4本）。盛りすぎない。
- 断定しすぎず、「次に欲しくなったら足す」という順番の発想を貫く。
